# 2026-03-09 Cron 推送问题修复

## 问题背景

早上 8:00 的 `morning-evolution-surprise` 和 9:00 的 `github-trending-daily` cron 任务消息推送异常。

## 诊断过程

### 1. 检查 cron 任务状态

```bash
openclaw cron list --json | jq '.jobs[] | {name, lastStatus, lastError}'
```

发现：
- `github-trending-daily` 没有 `agentId`
- `lastStatus: "error"`, `lastError: "⚠️ ✉️ Message failed"`

### 2. 对比 laoda 配置

```bash
ssh root@119.91.63.250 "cat /root/.openclaw/cron/jobs.json | jq '.'"
```

发现关键差异：
- laoda: `"to": "c2c:B0327EA1..."`
- violet: `"to": "qqbot:c2c:FD54D50..."`

### 3. 检查 Gateway 日志

```bash
grep "Message failed\|announce" /tmp/openclaw/openclaw-2026-03-09.log
```

发现：
- 消息实际发送成功（QQ API 200 OK）
- 但系统报告 "Message failed"

## 修复内容

### 1. 添加 agentId

```bash
openclaw cron edit 062e7113-fb05-4677-a7cc-3c0167b13f9d --agent violet
```

### 2. 修正 delivery.to 格式

```bash
openclaw cron edit <job-id> --to "c2c:FD54D50FDB86B669EC3195C21BFA0FBB"
```

修复的任务：
- `github-trending-daily`
- `morning-evolution-surprise`
- `daily-memory-review`

### 3. 重启 Gateway

```bash
openclaw gateway restart
```

## 发现的 OpenClaw Bug

### Bug 1: Usage Tokens 显示 0

- **原因**：自定义 provider 的 `prompt_tokens`/`completion_tokens` 未映射到 `input`/`output`
- **影响**：`/status` 始终显示 `Context: 0/98k (0%)`
- **状态**：已提交 GitHub Issue #40424

### Bug 2: Cron Announce Delivery 不工作

- **原因**：cron 子会话的 `announce` 绑定为 `null`
- **现象**：消息无法直接发送，需通过 `subagent_announce` 事件 + 主会话转发
- **状态**：已补充到 GitHub Issue #40424

- 09:51 再次触发 → ✅ 消息送达
