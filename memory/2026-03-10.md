## Context Engine 压缩问题修复

### 问题现象
- laoda 服务器上下文达到 180k/200k (90%) 时
- 出现 `Queue: collect (depth x)` 触发 block streaming
- 消息卡死，压缩超时
- 重启 gateway 后无法断电续接

### 根本原因
1. `ownsCompaction: false` — 插件被动等待 OpenClaw 调用压缩
2. `compaction.mode: "safeguard"` — 只预留空间，不主动调用 LLM 压缩
3. 触发太晚 — 到 90% 才尝试压缩，上下文太大导致超时
4. 日志显示 `Compactions: 0` — 压缩从未成功执行

### 修复方案 (memory-lancedb-context v1.8.0)

**代码修改**：
```typescript
// 1. 让插件主动控制压缩
readonly info: ContextEngineInfo = {
  id: "memory-lancedb-context",
  version: "1.8.0",
  ownsCompaction: true, // 之前是 false
};

// 2. 新增配置项
enableProactiveCompaction: true,      // 启用主动压缩
autoCompactionThreshold: 0.65,        // 65% 触发
minTurnsBeforeCompaction: 10,         // 最少 10 条消息
compactionTimeoutMs: 120000,          // 2 分钟超时保护

// 3. 在 afterTurn 中检测并触发
if (usageRatio >= this.config.autoCompactionThreshold) {
  await this.compact({ ... });
}
```
