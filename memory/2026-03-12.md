# 2026-03-12

- 新机 `openclaw-violet`（Ubuntu 22.04.1 / Node 22.22.0）排障：先确认 OpenClaw CLI 为 2026.2.9，但 user gateway service 仍指向旧的 2026.1.29 入口；通过官方 `openclaw gateway uninstall/install/start` 重装 service 后，gateway unit 已对齐到当前安装版本并恢复正常探活。
- 后续尝试把新机升级到 OpenClaw 2026.3.8 时，最初 `npm i -g openclaw@2026.3.8` 报 ETARGET；继续排查发现不是包不存在，而是目标机 `systemd-resolved` 没有有效上游 DNS，导致 `registry.npmjs.org` 解析失败。
- 已在目标机添加 `/etc/systemd/resolved.conf.d/99-openclaw-dns.conf`（公共 DNS + fallback），重启 `systemd-resolved` 后，`registry.npmjs.org` 解析恢复，`npm view openclaw version` / `dist-tags` 已能看到 `2026.3.8` / `2026.3.11-beta.1`。
- OpenClaw 2026.3.8 全局 npm 安装已开始并拉到主包；安装过程中 `/usr/bin/openclaw` 暂时消失，但 `/usr/lib/node_modules/openclaw/openclaw.mjs --version` 已能返回 `OpenClaw 2026.3.8`。当时 gateway systemd 进程仍保持 active/running，说明未出现立刻掉服。
