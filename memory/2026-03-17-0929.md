## 2) OpenClaw 侧：加一个本地 provider（OpenAI-compatible）
核心就三件事：
- `baseUrl`: `http://127.0.0.1:1234/v1`
- `apiKey`: 随便填（LM Studio 通常不校验，但 OpenClaw 需要这个字段）
- `models[]`: 填你 `/v1/models` 里看到的 `id`

示例（照着改成你的模型 id）：

```json5
{
  "agents": {
    "defaults": {
      "model": { "primary": "lmstudio/your-model-id" },
      "models": {
        "lmstudio/your-model-id": { "alias": "Local" }
      }
    }
  },
  "models": {
    "mode": "merge",
    "providers": {
      "lmstudio": {
        "baseUrl": "http://127.0.0.1:1234/v1",
        "apiKey": "lmstudio",
        "api": "openai-responses",
        "models": [
          {
            "id": "your-model-id",
            "name": "Local Model",
            "reasoning": false,
            "input": ["text"],
            "contextWindow": 131072,
            "maxTokens": 8192,
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 }
          }
        ]
      }
    }
  }
}
```

**稳定性建议（很关键）：**
- `api: "openai-responses"`：更利于把“思考/最终输出”分离，少遇到 `<think>...</think>` 这类污染（你之前也碰到过类似问题）。
- `contextWindow / maxTokens`：别瞎填太大，按模型真实能力来；填虚高会导致莫名其妙的报错或超时。

## 3) “稳定优先”的补强：保留一个云端 fallback
本地再稳也会遇到：显存抖动、模型被卸载、LM Studio 服务重启等。建议在 OpenClaw 里做混合（本地优先、云端兜底）：

```json5
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "lmstudio/your-model-id",
        "fallbacks": ["anthropic/claude-sonnet-4-5"] // 举例，换成你实际可用的
      }
    }
  },
  "models": { "mode": "merge" }
}
```
