1. [fact] ## auto-updater.sh 执行链路

###触发方式
使用 **systemd user timer**（不依赖 OpenClaw cron）：
- Timer: `openclaw-auto-update.timer`
- Service: `openclaw-auto-update.service`
- 调度: 每天 04:00 Asia/Shanghai

### 执行链路
```
systemd timer (04:00)
    ↓
systemd service (oneshot)
    ↓
bash /root/.openclaw/workspace/scripts/auto-updater.sh
    ↓
更新: CLIProxyAPI / Management Center / OpenClaw
    ↓
日志: /root/.openclaw/logs/auto-update.log
```

### 手动触发
```bash
systemctl --user start openclaw-auto-update.service
```

### 配置位置
- Timer: `/root/.config/systemd/user/openclaw-auto-update.timer`
- Service: `/root/.config/systemd/user/openclaw-auto-update.service`
- 脚本: `/root/.openclaw/workspace/scripts/auto-updater.sh`

### 两节点状态
| 节点 | Timer 状态 |
|------|-----------|
| Violet | ✅ active |
| laoda | ✅ active |

### 脚本功能
- `update_cliproxyapi`: 从 GitHub releases 更新 CLIProxyAPI
- `update_management_center`: 更新管理面板 HTML
- `update_openclaw`: 更新 OpenClaw（备份→安装→重启→健康检查）
- `auto_slim`: 清理 npm cache (>1GB) + 旧备份 (保留1份)
- `final_recover`: 终态兜底，确保 gateway 始终运行

### 关键配置
- `UPDATE_STRATEGY="patch"`: 只更新补丁版本
- `BACKUP_KEEP=1`: 保留最近 1 份备份
- `NPM_TIMEOUT=1200`: npm 安装超时 20 分钟 — status=inferred
