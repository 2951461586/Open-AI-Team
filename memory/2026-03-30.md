### JWT 续期/刷新：不可能自动化
- 探了 30+ 候选端点（/auth/refresh, /token/refresh, /api/v1/auth/refresh 等）→ 全部 404
- 整个 `/api/v1/*` 路由被 JWT 中间件保护，返回 "Request denied by JWT Auth check. JWT is missing."
- 前端登录走 Cloudflare Turnstile 验证码 + SPA 渲染（CF_TURNSTILE_SITEKEY 已知）
- **结论: 没有公开的 refresh/login REST API，无法纯 API 续期**

### 自动化方案评估
| 方案 | 可行性 | 备注 |
|---|---|---|
| 纯 API refresh | ❌ | 端点不存在 |
| Headless browser 模拟登录 | ⚠️ 高风险 | 需过 Turnstile，随时可能失效 |
| 半自动（人工登录 + 告警 + 简化更新流程） | ✅ 推荐 | 已有基础设施支撑 |

### 决策
- 维持半自动方案：巡检脚本监控 JWT 剩余有效期，提前告警，人工更新。
- 不投入 headless browser 自动登录。

## 16:09 – 本机 LLM-API-Key-Proxy 路由策略确认

- 本机 MuleRun 代理当前路由：`ROTATION_MODE_MULERUN=balanced` + `MAX_CONCURRENT_REQUESTS_PER_KEY_MULERUN=1`
- 代码默认 `DEFAULT_ROTATION_TOLERANCE=3.0`，因此 balanced 模式下实际 selection mode 为 **weighted-random**，不是严格轮询
- 选择顺序：先过滤 cooldown / model_cooldown / fair-cycle exhausted，再优先完全空闲 key（Tier 1）；若可复用并发则才进 Tier 2
- weighted-random 权重公式：`(max_usage - usage) + tolerance + 1`；低使用 key 权重更高，但保留随机性
- 当前单 key 单并发，因此 Tier 2 基本不参与；整体效果是**低使用优先的随机均衡轮转**
