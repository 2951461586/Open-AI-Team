## Mulerun Proxy Anthropic Bare Model KeyError 修复（12:30）

### 根因
- mulerun-proxy 的 Anthropic Messages 流式路径 (`/v1/messages`) 收到裸 model `claude-opus-4-6` 时，内部按 `model.split("/")[0]` 解析 provider，把裸名误当 provider，最终抛 `KeyError('claude-opus-4-6')`。
- OpenClaw 的 `anthropic-messages` adapter 会按 provider model id 发送请求，本机 mulerun provider 的 model id 之前配成裸名，导致每次走 Anthropic 路径都触发此 bug。
- 非流式 OpenAI-compat 路径不受影响（那条路走 `mulerun/claude-opus-4-6` full ref）。

### 修复内容
1. **mulerun-proxy 代码补丁**：`src/rotator_library/client.py` 的 `anthropic_messages()` 入口新增 bare model 自动标准化逻辑——如果 model 无 `/`，查 `model_definitions` 唯一匹配的 provider 后自动补前缀。
2. **OpenClaw 配置最终态 apply**：
   - `models.providers.mulerun.models[0].id` → `mulerun/claude-opus-4-6`（从裸名改为 full ref）
   - `agents.defaults.models` 删除裸 key `claude-opus-4-6`，只保留 `mulerun/claude-opus-4-6`
   - `fallbacks` 继续保持 full ref

### 验收
- 直打 proxy `/v1/messages` + 裸 model `claude-opus-4-6` → 200 OK / pong ✅
- OpenClaw 配置面无裸 model 尾巴 ✅
- gateway 已重载生效 ✅

- laoda / lebang 两侧尚未同步此次 full-ref 收口
