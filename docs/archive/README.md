# Archive

This directory contains historical, retired, and no-longer-primary documents.

## What belongs here

- completed migration/phase plans
- retired compatibility explanations
- historical deployment maps
- old boundary investigations that are no longer current authority
- pre-productization notes kept for provenance only

## What does not belong here

- current product definitions
- current authority docs
- current onboarding guidance
- current release engineering guidance

## How to use archive docs

Use archive docs for:
- provenance
- design history
- migration archaeology
- validating why a decision changed
- completed phase/migration plan lookup under `migration-history/`

Do **not** use archive docs as the default source of truth for current product behavior.
Do **not** cite archive docs as current authority unless you have separately verified the live owning surface.
If an archive document conflicts with `README.md`, `GETTING-STARTED.md`, `docs/oss/repo-authority.md`, or current package/app-owned code, the non-archive surface wins.
